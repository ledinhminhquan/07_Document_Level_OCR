# Data Card — P07 Document-Level OCR: Post-OCR Correction Corpus

> HuggingFace-style dataset card for the data used to train and evaluate the **trainable
> differentiator** of the P07 Document-Level OCR system: a **Post-OCR Error Correction**
> model. The corpus pairs **noisy OCR text** with **clean gold text** and is built from
> two complementary sources — a reproducible **synthetic OCR-noise generator** and the
> **real PleIAs Post-OCR-Correction** corpus.

- **Author:** Le Dinh Minh Quan (student 23127460)
- **Course:** NLP in Industry — final assignment
- **Package:** `src/dococr/` (data layer: `data/ocr_noise.py`, `data/corpus.py`, `data/dataset.py`, `data/download_dataset.py`)
- **Language:** English (`en`)
- **Task:** Post-OCR Error Correction — sequence-to-sequence `text -> text`

---

## 1. Dataset Summary

The trained ML core of P07 is **not** the OCR engine and **not** the layout/reading-order
logic (those are pretrained/algorithmic). It is a **Post-OCR Error Correction** model
(`google/byt5-small`) that takes the error-riddled output of *any* OCR engine and emits
cleaner text, measurably lowering Character Error Rate (CER) and Word Error Rate (WER).

To train and evaluate that corrector, this corpus provides aligned **(noisy, clean)** text
pairs from two sources:

| Source | Role | Origin | Rows |
|---|---|---|---|
| **Synthetic** | PRIMARY | Reproducible OCR-noise generator (`src/dococr/data/ocr_noise.py`) corrupting clean English text | train 60,000 / val 4,000 / test 4,000 (defaults) |
| **PleIAs/Post-OCR-Correction** (config `english`) | Real mix + real eval slice | Hugging Face dataset (CC0-1.0) | 31,300 |
| **FrancophonIA/ICDAR_2019_Competition_Post-OCR_Text_Correction** | Optional benchmark | ICDAR 2019 manual post-OCR corpus | viewer disabled; loader degrades gracefully |

The two sources are complementary: the **synthetic** generator gives unlimited, fully
controllable, reproducible noise (tunable error rate, no licensing friction), while the
**real PleIAs** corpus grounds the model in actual OCR error distributions and supplies a
real-world evaluation slice that guards against the sim-to-real gap.

---

## 2. Supported Task and Leaderboard

- **Task:** Post-OCR Error Correction (text-to-text seq2seq).
- **Model input:** noisy OCR text, prefixed with `"correct: "`.
- **Model target:** clean / corrected gold text.
- **Headline metric:** **error reduction** — `% reduction = (CER_before − CER_after) / CER_before`, where `CER_before = CER(raw_OCR, gold)` and `CER_after = CER(corrected, gold)`. Same formulation for WER.
- **Safety gate:** fraction of examples **improved vs. degraded vs. unchanged**. A corrector that fixes 40% but breaks 30% is useless; `degraded << improved` is required.
- **Baseline to beat:** **identity** (raw OCR, no correction) — the canonical post-OCR baseline. Optional comparators: SymSpell dictionary corrector (`symspellpy`, MIT) and `oliverguhr/spelling-correction-english-base` (MIT BART).

## 4. Dataset Structure

### 4.1 Data fields (the (noisy, clean) pair schema)

Every example is a single aligned pair. Field names are normalized across sources at load
time:

| Field | Type | Description |
|---|---|---|
| `noisy` / `text` | string | OCR-degraded input text (the model input). In PleIAs this is the `text` column (noisy OCR). |
| `clean` / `corrected_text` | string | Gold reference text (the model target). In PleIAs this is the `corrected_text` column. |

At training time the input is wrapped with the prefix `"correct: "`:

```
input  = "correct: " + noisy
target = clean
```

### 4.2 Data splits

| Split | Synthetic (default) | Notes |
|---|---|---|
| `train` | 60,000 | Generated from clean English text + OCR-noise model |
| `validation` | 4,000 | Used for early stopping on CER |
| `test` | 4,000 | Held-out synthetic noise distribution |
| **Real eval slice** | — | Drawn from PleIAs `english` (31.3K rows) to report real-world CER alongside synthetic |

**Leakage control:** splits are made **leakage-free by de-duplicating on the *clean* text**,
so no clean sentence appears across train/val/test. The synthetic sizes above are
configurable defaults.

## 5. The Synthetic OCR Confusion Model

The primary data is generated by `src/dococr/data/ocr_noise.py`, which corrupts clean
English text with a realistic OCR confusion model at a **tunable character error rate
(`char_error_rate`, default `0.08`)**.

### 5.1 Error types covered

| Error type | Description | Example |
|---|---|---|
| **Character substitution** | Visually-confusable character swaps from curated confusion sets | `rn ↔ m`, `cl ↔ d`, `O ↔ 0`, `l ↔ 1`, `e ↔ c`, `S ↔ 5`, `g ↔ q`, … |
| **Insertion** | Spurious extra characters | `the` → `thfe` |
| **Deletion** | Dropped characters | `correction` → `corection` |
| **Word merge / split** | Spacing corruption — words run together or split apart | `the cat` → `thecat`; `correction` → `corre ction` |
| **Case flip** | Random case inversion | `Document` → `doCument` |

These mirror the dominant failure modes of real printed-text OCR: confusable glyphs
(`rn`→`m`, `0`↔`O`, `1`↔`l`), and merged/split words from imperfect word segmentation.
Because the corrector is byte/char-level (ByT5), it is well matched to this char-level noise.

### 5.2 Anti-overfitting / generalization

A **diverse confusion model**, a **mix of real PleIAs data**, early stopping, and weight
decay are used together so the corrector does not memorize a single synthetic noise pattern.
Noise severity is also varied for robustness evaluation (a robustness report sweeps across
`char_error_rate` levels).

## 7. Validated Baseline Difficulty

On the **synthetic test distribution**, the **identity baseline** (raw OCR, no correction)
measures:

| Metric (identity / raw OCR) | Value |
|---|---|
| CER | ~0.088 |
| WER | ~0.49 |
| Exact-match | ~0.0005 |

This confirms the corrector has a **clear, measurable job**: nearly every example is wrong
at the word level, and exact-match is effectively zero before correction. The trained
**ByT5** corrector reduces CER substantially below this identity baseline.

## 8. Intended Use

- **Primary:** train and evaluate a post-OCR correction model that lowers CER/WER on top of
  any OCR front-end (Tesseract, doctr, PaddleOCR, Surya, TrOCR, EasyOCR, …) in the P07
  document pipeline.
- Provide a **reproducible** training signal (synthetic generator with fixed seeds and a
  controllable error rate) plus a **real-world grounding/eval slice** (PleIAs `english`).
- Support **robustness analysis** by regenerating data at varying noise severities.
- Benchmark against the identity baseline and optional dictionary/neural comparators.

This corpus is intended for **research and the coursework submission**, not as a general
standalone NLP dataset release.

---

## 9. Out-of-Scope Use

- **Non-English text.** The corpus is English-only; using it for other languages is out of
  scope (PleIAs has other configs, but this card covers `english` + English synthetic only).
- **Handwriting-specific noise.** The synthetic confusion model targets **printed-text** OCR
  errors (confusable glyphs, merged/split words). It does **not** model handwriting-specific
  degradation, ink bleed, stroke variation, or cursive segmentation. The pipeline can route
  to handwriting recognizers (e.g. `microsoft/trocr-*-handwritten`), but this *training
  corpus* is not built for handwriting error distributions.
- **Layout / reading-order ground truth.** This corpus is text-pair only; document layout
  and reading-order evaluation use separate datasets (`pierreguillou/DocLayNet-base`,
  `FUNSD`, `PubLayNet`) and are not part of this data card.
- **Engine-exact error matching.** Synthetic noise is a *realistic approximation*, not a
  replay of a specific OCR engine's exact error distribution (mitigated by also reporting
  real PleIAs CER and mixing real data).

## 11. Known Biases and Limitations

- **Sim-to-real gap.** Synthetic noise may not match a specific OCR engine's true error
  distribution. Mitigation: mix real PleIAs data into training and **report real PleIAs CER**
  alongside synthetic metrics.
- **English-first bias.** Vocabulary, confusion sets, and gold text are English; the model
  will not generalize to other scripts or languages.
- **Printed-text bias.** Confusion sets are tuned for printed OCR, not handwriting, dense
  tables, figures, or heavily rotated/skewed scans.
- **Corrector-makes-it-worse risk.** A neural corrector can *hallucinate* or over-rewrite.
  The data/eval design guards against this with the **% improved vs. degraded safety gate**,
  and at inference the agent applies a **bounded-edit acceptance gate** (edit ratio `<= 0.35`,
  else keep raw OCR). A corrector that improves few and degrades many fails the gate.
- **PII / privacy.** Real OCR corpora and uploaded documents can contain PII (names, IDs,
  addresses, financial data). The pipeline minimizes retention/logging of raw images and
  text and offers an on-prem/no-retention option with TTL cleanup. OCR of private or
  copyrighted documents requires appropriate rights/consent.
- **Optional-source fragility.** The ICDAR 2019 benchmark has its dataset viewer disabled;
  the loader is written to degrade gracefully if it cannot be fetched.

## 12. Dataset Creation

- **Synthetic:** clean English text is passed through `ocr_noise.py`, which applies the
  confusion model (Section 5) at the configured `char_error_rate`. Splits are produced with
  clean-text de-duplication to prevent leakage. Default sizes: train 60,000 / val 4,000 /
  test 4,000.
- **Real:** the PleIAs `english` config (31.3K rows, columns `text` = noisy OCR,
  `corrected_text` = gold) is loaded via `data/download_dataset.py` / `data/corpus.py`,
  field-normalized to the `(noisy, clean)` schema, and used both as a training mix and as a
  real-world evaluation slice.
- **Optional:** the ICDAR 2019 manual corpus may be loaded as an external benchmark.
